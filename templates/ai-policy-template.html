1. Purpose

This policy establishes guidelines for the responsible, secure, and effective use of Artificial Intelligence (AI) tools by employees, contractors, and affiliates of [Company Name]. It aims to mitigate risks related to data privacy, intellectual property, and ethical considerations.

2. Scope

This policy applies to:
* All employees, contractors, and third-party partners.
* All AI tools used for business purposes, including but not limited to: Large Language Models (LLMs) (e.g., ChatGPT, Claude, Gemini), image generators (e.g., Midjourney, DALL-E), and coding assistants (e.g., Copilot).
* Any AI features embedded within existing software (e.g., Zoom AI Companion, Microsoft 365 Copilot).

3. Approved Tools

The following AI tools are explicitly approved for use. Use of any other tool requires prior approval from IT/Security.

Tool Name Approved Use Case Data Classification Level
Microsoft Copilot (Enterprise) Drafting emails, summarizing internal docs Confidential / Internal
ChatGPT (Free/Plus) Ideation, public data analysis Public Data ONLY
GitHub Copilot Code generation, refactoring Internal Code
Midjourney Marketing images (non-commercial) Public

(Note: "Free" tiers of public tools often use your data for training. Treat them as PUBLIC environments.)

4. Prohibited Uses

Users are strictly prohibited from using AI for:
1. uploading Personally Identifiable Information (PII) (e.g., SSNs, addresses, health data) into any public AI tool.
2. Generating or manipulating content for malicious purposes (e.g., deepfakes, phishing emails).
3. Making automated decisions that legally or significantly impact individuals (e.g., hiring, lending) without human review.
4. Copyright Infringement: Generating content that knowingly mimics a specific artist or creator's style for commercial gain without license.
5. Bypassing Security Controls: Using AI to write malware or exploit vulnerabilities.

5. Data Handling Guidelines

  • Public Data: Safe for all approved tools.
  • Internal Data: Safe for Enterprise-tier tools (e.g., Copilot Enterprise, ChatGPT Team) where data training is disabled.
  • Confidential/PII: NEVER put into public AI tools. Only allowed in isolated, approved environments (e.g., private Azure OpenAI instance).
  • Source Code: Do not paste proprietary code into public chatbots. Use approved coding assistants only.

6. Human-in-the-Loop (Review Requirement)

AI outputs are often inaccurate ("hallucinations"). All AI-generated work MUST be reviewed by a human expert before publication, deployment, or decision-making. The user is accountable for the final output, regardless of its origin.

7. Incident Reporting

If you suspect an AI tool has leaked data, generated harmful content, or been used unethically, report it immediately to [Security Email/Slack Channel].

8. Acknowledgement

By using AI tools at [Company Name], I agree to adhere to this policy.
