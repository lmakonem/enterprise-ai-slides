The Big Three: ChatGPT, Claude & Gemini

Let's start with the three dominant commercial LLM platforms. Each has distinct strengths that map to different use cases. Understanding these differences is essential for informed procurement.

The AI Tools Landscape (2025)

This landscape changes quarterly, but the categories are stable: closed-source frontier models, open-source models, code assistants, and specialized tools. We'll cover each category. The key takeaway is that no single tool wins at everything ‚Äî tool selection is about matching capability to use case.

Code Assistant Security Considerations

Risks

  • ‚ö†Ô∏è Code exfiltration: Your proprietary code sent to model provider
  • ‚ö†Ô∏è Insecure suggestions: Stanford study found Copilot generates vulnerable code in 40% of security-sensitive scenarios
  • ‚ö†Ô∏è License contamination: AI may reproduce GPL-licensed code patterns
  • ‚ö†Ô∏è Secret leakage: Models can suggest hardcoded API keys from training data

Mitigations

  • ‚úÖ Use enterprise tiers with data privacy guarantees
  • ‚úÖ Enable content exclusion filters for sensitive repos
  • ‚úÖ Run SAST/DAST on all AI-generated code
  • ‚úÖ Mandate code review for AI-assisted commits
  • ‚úÖ Consider on-premise alternatives (Tabnine, self-hosted)
  • ‚úÖ Train developers to verify, not blindly accept
This slide is critical for security teams. The Stanford study on Copilot and security is sobering ‚Äî AI code assistants can introduce vulnerabilities that look plausible. The solution isn't to ban AI assistants; it's to integrate them into your existing secure development lifecycle. All AI-generated code should go through the same review and scanning as human-written code.

Self-Host vs. Cloud: The Economics

Self-Host When:

  • ‚úÖ Data cannot leave your infrastructure
  • ‚úÖ Volume exceeds ~$10K/month in API costs
  • ‚úÖ You need to fine-tune on proprietary data
  • ‚úÖ Latency requirements are strict (<100ms)
  • ‚úÖ You want zero vendor dependency

Use Cloud APIs When:

  • ‚úÖ You need frontier quality (GPT-4, Claude Opus)
  • ‚úÖ Volume is low-to-moderate
  • ‚úÖ You want zero infrastructure management
  • ‚úÖ You need rapid iteration / prototyping
  • ‚úÖ Vendor DPA meets your compliance needs

üí∞ Break-even example: Running Llama 3 70B on 4x A100 GPUs costs ~$8K/month (cloud GPU). At GPT-4o API rates, that equivalent throughput would cost ~$25K/month. Self-hosting saves ~$17K/month if you have the engineering team to maintain it.

The economics are clear for high-volume use cases, but don't underestimate operational costs. You need GPU expertise, monitoring, failover, security patching, and model updates. Many enterprises start with cloud APIs and migrate high-volume workloads to self-hosted as they scale. The hybrid approach is often optimal.

Evaluation Framework: How to Choose

Now let's build a systematic framework for evaluating AI tools. Ad-hoc "this one feels better" doesn't work for enterprise procurement. You need measurable criteria.

The ACCPS Evaluation Framework

A ‚Äî Accuracy

How correct and reliable are outputs?

  • Factual correctness
  • Hallucination rate
  • Consistency across runs

C ‚Äî Cost

Total cost of ownership?

  • Per-token pricing
  • Seat licensing
  • Infrastructure costs

C ‚Äî Compliance

Does it meet regulatory needs?

  • Data residency
  • SOC 2 / ISO 27001
  • GDPR / HIPAA

P ‚Äî Performance

Speed and throughput?

  • Latency (time to first token)
  • Tokens per second
  • Rate limits / quotas

S ‚Äî Security

Is your data protected?

  • Data processing agreement
  • Training data opt-out
  • Prompt injection resistance
ACCPS gives you a structured way to evaluate any AI tool. Score each dimension 1-5, weight by your organization's priorities, and compare. A healthcare company might weight Compliance and Security at 5x while a startup weights Cost and Performance. This framework is in your lab worksheet ‚Äî you'll use it to evaluate tools hands-on.

Code: Comparing Models Programmatically

import openai, anthropic, time

prompt = "Explain zero-trust architecture in 3 sentences for a CISO."

# OpenAI
oai = openai.OpenAI(api_key="sk-...")
start = time.time()
gpt_resp = oai.chat.completions.create(
    model="gpt-4o", messages=[{"role":"user","content":prompt}], temperature=0.2
)
gpt_time = time.time() - start

# Anthropic
ant = anthropic.Anthropic(api_key="sk-ant-...")
start = time.time()
claude_resp = ant.messages.create(
    model="claude-3-5-sonnet-20241022", max_tokens=300,
    messages=[{"role":"user","content":prompt}]
)
claude_time = time.time() - start

print(f"GPT-4o ({gpt_time:.1f}s): {gpt_resp.choices[0].message.content}")
print(f"Claude ({claude_time:.1f}s): {claude_resp.content[0].text}")
print(f"GPT tokens: {gpt_resp.usage.total_tokens} | Claude tokens: {claude_resp.usage.input_tokens + claude_resp.usage.output_tokens}")
This pattern ‚Äî same prompt, multiple models, timed ‚Äî is exactly how you should evaluate models for your specific use cases. Generic benchmarks tell you about average performance; this tells you about YOUR performance on YOUR prompts. In the lab, you'll extend this to include Gemini and score outputs on accuracy, format, and helpfulness.
