üìù Step-by-Step Instructions

Part 1: Account & Token Setup (10 min)

  1. Create Account: Go to huggingface.co and sign up. verify your email.
  2. Get Access Token:
    • Click your profile picture -> Settings -> Access Tokens.
    • Click Create new token.
    • Name it: lab-token.
    • Permissions: Read.
    • Copy the token (starts with hf_...). Save it somewhere safe.

Part 2: Model Scouting (20 min)

Imagine you are an enterprise architect. You need to find a model for Sentiment Analysis (classifying text as Positive/Negative).
1. Search: Go to the Model Hub. Search for "Sentiment Analysis."
2. Filter: Select "Text Classification" on the left.
3. Evaluate: Pick 3 models (e.g., distilbert-base-uncased-finetuned-sst-2-english, cardiffnlp/twitter-roberta-base-sentiment).
4. Rubric: Fill out the Model Evaluation Rubric for the most promising one. Check its license and limitations.

Part 3: Using the Inference API (20 min)

Instead of downloading huge models, we can use Hugging Face's servers via an API.

  1. Open Python: Use a local script or a Google Colab notebook.
  2. Install Library:
    bash pip install huggingface_hub
  3. Run the Script:
    Copy the code from app/inference_api_demo.py (provided in the lab files).
    Replace YOUR_HF_TOKEN with the token you created in Part 1.
    Run the script.
    • Challenge: Change the prompt variable to something related to your industry.

Part 4: Deploying a Space (30 min)

Now, the grand finale: deploying a web app. We will use Gradio, a library that turns Python functions into web UIs instantly.

  1. Create a New Space:

    • Go to Hugging Face -> Spaces -> Create new Space.
    • Space Name: my-first-summarizer
    • License: MIT
    • SDK: Gradio
    • Hardware: CPU Basic (Free)
    • Visibility: Public

  2. Add Files:
    You can edit files directly in the browser (easiest) or clone the repo.

    • Create app.py: Paste the code from app/app.py (in lab files).
    • Create requirements.txt: Paste the code from app/requirements.txt.

  3. Commit & Build:

    • Click "Commit changes to main."
    • Watch the "Building" status at the top. It will turn green ("Running") in 1-2 minutes.

  4. Test It:

    • Find a long news article.
    • Paste it into your new app.
    • Click "Submit."
    • Copy the URL and share it!

Part 5: Reflection (10 min)

  • How easy was it to deploy compared to setting up a traditional server?
  • What are the risks of using a public Space for company data? (Hint: Anyone can see it unless it's private).
