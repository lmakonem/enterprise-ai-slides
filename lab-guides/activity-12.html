Activity 12: Hands-On with MCP (Model Context Protocol)

Duration: 90 Minutes
Goal: Connect Claude Desktop to local files, databases, and a custom Python server to experience "AI Integration" firsthand.

Prerequisites

  • A computer (Mac, Windows, Linux)
  • Claude Desktop App installed
  • Python 3.10 or higher installed
  • A code editor (VS Code recommended)

Part 1: The "Magic Moment" â€” Connecting Local Files (20 min)

Objective: Give Claude read-access to a folder on your computer without uploading files manually.

  1. Open Claude Desktop App.

    • Go to Settings (or Claude > Settings on Mac).
    • Look for the Developer or MCP section. (Note: The location changes as the feature evolves. If you don't see it, check the "Feature Preview" menu).
    • Alternatively, you can edit the config file directly (see Part 3).

  2. Configure the Filesystem Server:

    • We will use the pre-built @modelcontextprotocol/server-filesystem.
    • You need npx (part of Node.js) installed for this standard server. If you don't have Node.js, skip to Part 3 where we use Python.
    • Command to add:
      bash npx -y @modelcontextprotocol/server-filesystem /Users/yourusername/Desktop/my-project
      (Replace the path with a real folder on your computer).

  3. Test It:

    • Create a file named secret_project.txt in that folder with the text: "The secret code is BLUEBERRY."
    • Open a new chat in Claude.
    • You should see a generic "hammer/wrench" icon or an attachment icon indicating MCP tools are active.
    • Ask: "Please read the file secret_project.txt in my project folder and tell me the code."
    • Observe: Claude didn't hallucinate. It actually read the file from your disk.

Part 2: The "Database Whisperer" (20 min)

Objective: Let Claude query a SQLite database to answer questions.

  1. Prepare the Data:

    • We have provided a sample_database.sql file in the app/ folder.
    • Create a SQLite database from it (or use the pre-made company.db if provided).
    • Command (if you have sqlite3 installed):
      bash sqlite3 company.db < app/sample_database.sql

  2. Add the SQLite Server:

    • We will use the official SQLite MCP server.
    • Add to your Claude Config:
      json "sqlite": { "command": "uvx", "args": ["mcp-server-sqlite", "--db-path", "/absolute/path/to/company.db"] }

  3. Test It:

    • Restart Claude Desktop.
    • Ask: "What is the most expensive product in the database?"
    • Ask: "Who is the highest paid employee in Engineering?"
    • Observe: Claude writes and executes SQL queries on the fly to answer you.

Part 3: Build Your Own MCP Server (40 min)

Objective: Write a Python server that exposes a custom "Company Directory" API.

Step 1: Setup

  1. Navigate to the app/ folder in your terminal.
  2. Create a virtual environment:
    bash python -m venv venv source venv/bin/activate # On Windows: venv\Scripts\activate
  3. Install the MCP SDK:
    bash pip install mcp

Step 2: Review the Code

Open server.py in VS Code. This file uses the FastMCP class to make creating servers easy.
- Resources: @mcp.resource() decorator makes data readable.
- Tools: @mcp.tool() decorator makes functions callable.
- Prompts: @mcp.prompt() decorator creates templates.

Step 3: Configure Claude

You need to tell Claude where your python script is.
1. Open your config file:
- macOS: ~/Library/Application Support/Claude/claude_desktop_config.json
- Windows: %APPDATA%\Claude\claude_desktop_config.json
2. Add your server:
json { "mcpServers": { "my-company-directory": { "command": "/absolute/path/to/your/venv/bin/python", "args": ["/absolute/path/to/your/app/server.py"] } } }
Tip: Use the absolute path to the Python executable inside your virtual environment!

Step 4: Test Your Creation

  1. Restart Claude Desktop.
  2. Look for the plug icon (MCP).
  3. Try the Resource: "Read the full employee list from the company directory."
  4. Try the Tool: "Find the email for the person who works in HR."
  5. Try the Prompt: "Use the 'new_hire_onboarding' prompt for Alice Johnson."

Step 5: Challenge

Modify server.py to add a new tool: calculate_bonus(salary).
- It should return 10% of the input salary.
- Restart Claude and ask it to calculate a bonus for Bob.

Troubleshooting

  • Claude shows "Connection Failed": Check your paths in the config JSON. JSON doesn't like backslashes \ on Windows unless escaped \\.
  • "Command not found": Ensure you are pointing to the python executable inside the virtual environment.
  • Logs: Check Claude's logs for error messages.
