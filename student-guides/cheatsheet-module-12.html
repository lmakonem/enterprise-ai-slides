What is MCP?

Model Context Protocol is an open standard that connects AI models (LLMs) to data and tools. It replaces custom integrations with a universal "plug-and-play" system.

Key Terminology

  • Client (Host): The AI application (Claude Desktop, Cursor, Zed) that consumes data.
  • Server: The process that provides data and capabilities (e.g., GitHub Server, Postgres Server).
  • Transport: The communication channel (Stdio for local, SSE for remote).
  • Resource: Passive data (read-only). Example: file:///logs/error.txt.
  • Tool: Actionable function (write/execute). Example: delete_file("log.txt").
  • Prompt: User-facing template. Example: summarize_bugs.

Architecture Diagram (Simplified)

[ User ] -> [ Host (Claude) ] <--> [ Transport (Stdio/SSE) ] <--> [ MCP Server ] <--> [ Data Source (DB/API) ]

Minimal Python Server Template

from mcp.server.fastmcp import FastMCP

mcp = FastMCP("My Server")

@mcp.resource("my://data")
def get_data() -> str:
    return "Hello World"

@mcp.tool()
def add(a: int, b: int) -> int:
    return a + b

if __name__ == "__main__":
    mcp.run()

Security Checklist

  • [ ] Least Privilege: Does the server have more access than it needs?
  • [ ] Human-in-the-Loop: Does the Host ask for confirmation before critical actions?
  • [ ] Sandboxing: Is the server running in a container or VM?
  • [ ] Input Validation: Does the tool validate arguments to prevent injection?
  • [ ] Audit Logging: Are tool executions logged?

Troubleshooting

  • "Connection Refused": Server crashed or port is blocked. Check logs.
  • "Tool Not Found": Restart the Host (Claude). Config changes require a restart.
  • "Permission Denied": Check file permissions or API keys.
  • Hallucination: Ensure tool descriptions are clear so the AI knows when to use them.
